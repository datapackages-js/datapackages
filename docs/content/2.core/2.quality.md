# Quality & Coverage

DataPackages is built with a strong focus on reliability. We use comprehensive testing to ensure that every part of the ecosystem works as expected.

## Testing Strategy

We use [Vitest](https://vitest.dev/) for unit and integration testing.

- **Unit Tests**: Verify individual functions and classes.
- **Integration Tests**: Verify how middleware interacts with datasets.
- **Mocking**: We use extensive mocking (e.g., for `IndexedDB`, `fetch`) to ensure tests are fast and deterministic.
