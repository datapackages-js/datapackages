# Contributor Guide

Welcome! This guide will help you get set up to contribute to the DataPackages monorepo.

## ğŸ“ Repository Structure

We use a monorepo structure managed by npm workspaces:

- `packages/dataset`: The core engine and middleware system.
- `packages/plugin-*`: Official plugins (cache, search, live-update, stream).
- `packages/*` (datasets): Individual data packages like `countries`.
- `docs`: The documentation site (Nuxt Content).

## ğŸš€ Getting Started

1. **Clone and Install**:
```bash
git clone https://github.com/your-repo/datapackages.git
cd datapackages
npm install
```

2. **Run Tests**:
We use Vitest for testing. You can run all tests from the root:
```bash
npm run test
```

To run tests for a specific package:
```bash
npm run test -w @datapackages/dataset
```

3. **Build**:
```bash
npm run build
```

## â• Adding a New Dataset

1. **Create package**: Create a new directory in `packages/`.
2. **Setup config**: Add a `datasets.config.ts` using `defineDatasets`.
3. **Fetch data**: Run `npx datapackage update-source` in your new package directory.
4. **Export service**: Create a class extending `Dataset<TItem>`.

4. Document the plugin in `docs/content/4.plugins/`.

## ğŸ“œ Coding Standards

- **TypeScript**: Everything must be strictly typed. Avoid `as any`.
- **Hooks**: Use the lifecycle hooks (`onInit`, `onLoad`, `dispose`) appropriately.
- **Documentation**: All new features must include JSDoc comments and an entry in the `docs/` folder.
- **Tests**: Core logic should aim for high test coverage.
